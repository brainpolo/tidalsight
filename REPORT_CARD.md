# Report Card

Asset-level scoring system that produces a holistic buy/sell/hold assessment across up to 6 dimensions, each scored 1–4, culminating in an overall verdict with a 12-month price target. Equities are assessed on all 6 dimensions; crypto, commodities, and currencies use 4 (Finance and People are equity-only). Scores are weighted and scaled to a consistent 0–30 range regardless of section count.

## Philosophy — Herzberg's Two-Factor Theory

The report card is structured around [Herzberg's Two-Factor Theory](https://en.wikipedia.org/wiki/Two-factor_theory), separating analysis into **Hygiene Factors** (baseline requirements that prevent failure) and **Motivators** (growth drivers that create upside).

**Hygiene Factors** — governed by external factors and public information (what the data shows):

| # | Section | What it answers | Asset classes | Weight |
|---|---------|-----------------|---------------|--------|
| 1 | **Finance** | Is the balance sheet solid? Revenue growing? Margins healthy? | Equities only | 1.5x |
| 2 | **Sentiment** | What does the market think? Is crowd sentiment aligned or divergent? | All | 1.5x |
| 3 | **Risk** | What macro, regulatory, or geopolitical threats exist? | All | 1.5x |

**Motivators** — governed by internal factors (where your thesis lives):

| # | Section | What it answers | Asset classes | Weight |
|---|---------|-----------------|---------------|--------|
| 4 | **Valuation** | Is the asset cheap or expensive relative to intrinsic value? | All | 1.0x |
| 5 | **Product** | Does the asset have compounding moats and network effects? | All | 1.0x |
| 6 | **People** | Is management capable, aligned, and executing well? | Equities only | 1.0x |

Hygiene factors are weighted 1.5x because they represent non-negotiable conditions — a company with great leadership but critical financial health still fails.

The **Overall Assessment** synthesises all scored sections into a verdict (Strong Buy / Buy / Hold / Sell / Strong Sell), a 12-month price target, key drivers, and key risks. For non-equity assets with 4 sections, raw scores are scaled up to the 0–30 range so verdict thresholds remain consistent.

## Personalisation — user notes as context

Motivator sections (Valuation, Product, People) can be personalised by user notes. When a user saves notes for an asset:

1. **Base scores** are generated by the agent using only public data
2. **Revision agents** take the base score + user notes and compute a delta adjustment (±0.0–0.5)
3. The **Personal** view shows the revised scores; the **Base** view shows the unmodified agent-only scores
4. A toggle in the Motivators header lets users switch between Personal and Base views
5. The Overall Assessment also recalculates based on the active view

This means two users looking at the same asset can see different scores if their investment theses differ — the system adapts to individual conviction.

## How scores are generated

Every section follows a **Celery fire-and-forget** pattern:

1. User visits the asset page → HTMX fires GET requests for each report card section
2. If a fresh cached score exists → return the scored card immediately
3. If stale or missing → enqueue a Celery task, return a loading skeleton
4. HTMX polls every 6s until the score lands
5. Distributed locks (`cache.add()`) prevent duplicate agent runs for the same asset

### Agent grounding

All agents receive standardised grounding context appended to their prompts via `analyst/grounding.py`:
- Current date for temporal awareness
- Response standards: direct tone, no hedging, present tense, institutional research quality

Grounding is **appended** (not prepended) to maximise KV-cache reuse when the same grounding block appears across calls with different lead-in content.

### Section details

#### Finance (equities only)
- **Agent:** `analyst/agents/financial_health_agent.py` — structured output (`FinancialHealthAssessment`: score 1–4, label, brief, strengths, concerns)
- **Data:** `Fundamental` model — revenue, revenue growth, earnings growth, profit margin, EPS, FCF, D/E, current ratio, ROE, dividend yield, market cap, beta, P/B
- **Prompt includes** S&P 500 benchmarks for context
- **Invalidation:** Fingerprint-driven. MD5 of all fundamental values compared against cached hash — regenerates only when data changes

#### Sentiment
- **Agent:** `analyst/agents/sentiment_agent.py` — structured output (score 1–4, label, brief, key_themes)
- **Data:** Reddit posts, HN posts, news articles (community posts)
- **Invalidation:** Source fingerprint (MD5 of post IDs) + 12-hour freshness TTL

#### Risk
- **Agent:** `analyst/agents/external_risk_agent.py` — structured output (score 1–4, label, brief, risk_factors)
- **Data:** Web search for macro, regulatory, geopolitical risks
- **Invalidation:** Time-based freshness TTL (2 weeks)

#### Valuation
- **Agent:** `analyst/agents/valuation_agent.py` — structured output (score 1–4, label, brief, bull_cases, bear_cases)
- **Data:** Fundamentals + web search for comparable valuations
- **Personalisation:** Revision agent adjusts base score with user notes/price target
- **Invalidation:** Fingerprint-driven (fundamentals hash) + 1-week freshness TTL

#### Product
- **Agent:** `analyst/agents/product_flywheel_agent.py` — structured output (score 1–4, label, brief, flywheel_strengths, moat_risks)
- **Data:** Web search for business model analysis
- **Personalisation:** Revision agent adjusts base score with user notes
- **Invalidation:** Time-based freshness TTL (1 week)

#### People (equities only)
- **Agent:** `analyst/agents/leadership_agent.py` — structured output (score 1–4, label, brief, people_strengths, people_risks)
- **Data:** Web search for management team, talent, and culture analysis
- **Personalisation:** Revision agent adjusts base score with user notes
- **Invalidation:** Time-based freshness TTL (1 week)

#### Overall Assessment
- **Agent:** `analyst/agents/overall_assessment_agent.py` — structured output (target_price, target_rationale, justification, key_drivers, key_risks)
- **Data:** All scored section results + their briefs, fed as context
- **Trigger:** Only runs after all applicable sections have scores (6 for equities, 4 for others)
- **Invalidation:** Composite fingerprint of all section scores

## Company Description

Each asset gets a 1-2 sentence Bloomberg-style description generated by a dedicated LLM agent with web search:
- **Agent:** `analyst/agents/description_agent.py`
- **Manager:** `analyst/managers/description_manager.py`
- **Storage:** Persisted to `Asset.description` in the database (not just cache)
- **Freshness:** Regenerates after 90 days (`DESCRIPTION_FRESHNESS_DAYS`)
- **Trigger:** Generated on asset creation and lazy-loaded on page visit

## Architecture

```
User visits asset page
  │
  ├─ HTMX GET /assets/{ticker}/report-card/{section}/
  │    ├─ Cache hit + fingerprint match → return scored card
  │    └─ Cache miss or fingerprint mismatch
  │         ├─ Enqueue Celery task (fire-and-forget)
  │         ├─ Return loading skeleton
  │         └─ HTMX polls every 6s until score lands
  │
  ├─ Personalisation (Motivator sections only)
  │    ├─ Base score cached per-asset
  │    ├─ Revision delta cached per-user-per-asset
  │    └─ Personal/Base toggle re-fetches with ?view=base
  │
  ├─ Overall Assessment
  │    ├─ Waits for all applicable sections (6 equity, 4 others)
  │    └─ Synthesises into verdict + price target
  │
  └─ Company Description
       ├─ Served from DB if fresh (<90 days)
       └─ Background Celery task if stale/missing
```

## Visual design

Report cards render in a 3-column CSS grid with equal-height cards. Each scored card has:
- Compact score/label header with staff-only regenerate button
- Structured factor rows with subtle copper/muted vertical bar accents
- Factors capped at 5 items per list (`|slice:":5"`)
- Bottom section pushed down with `mt-auto` so cards align regardless of content length
- Processing banner that auto-hides once all sections complete
